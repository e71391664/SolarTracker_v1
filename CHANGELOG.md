## [1.1.0] - 2026-06-06

### Added (Добавлено)

#### 📡 WiFi модуль и веб-интерфейс
- **WebServer** - встроенный HTTP сервер на базе ESP32
- **Access Point (AP) режим** - создание собственной WiFi сети
  - SSID: `SolarTracker`
  - Пароль: `12345678`  
  - IP адрес: `192.168.4.1`
- **Файлы**: 
  - `include/webserver.h` - заголовочный файл
  - `src/webserver.cpp` - реализация (350+ строк с HTML/CSS/JS)

#### 🌐 REST API
- `GET /` - главная страница HTML интерфейса
- `GET /api/status` - JSON с текущими данными (напряжения панелей, режим, статус мотора)
- `POST /api/cmd/left` - команда поворота влево
- `POST /api/cmd/right` - команда поворота вправо
- `POST /api/cmd/mode` - переключение режима AUTO/MANUAL

#### 🎨 Веб-интерфейс
- Адаптивный HTML5 интерфейс с градиентом фона
- Три кнопки управления: LEFT, RIGHT, AUTO/MANUAL
- Real-time мониторинг напряжений обеих панелей
- Визуализация разницы напряжений (Difference Meter)
- Индикаторы: статус мотора, текущий режим, время обновления
- Auto-refresh каждые 500 мс
- Поддержка мобильных устройств (responsive design)

#### 📚 Документация
- `WIFI_INTERFACE.md` - полная документация пользователя
- `ARCHITECTURE.md` - техническая архитектура и спецификация
- `examples_python.py` - Python клиент с примерами (SolarTrackerClient класс)
- `examples_javascript.js` - JavaScript клиент с примерами

### Changed (Изменено)

#### Integration
- `src/main.cpp` - добавлены инклюд `#include "webserver.h"` и вызовы:
  - `initWebServer()` в `setup()`
  - `handleWebServer()` в начало `loop()` (приоритетная обработка)
- `platformio.ini` - добавлены зависимости: `WiFi`, `WebServer`

### Technical Details (Технические детали)

#### Производительность
- Потребление памяти: +40 KB (HTML в PROGMEM)
- Задержка обработки: <10 мс
- Real-time обновление: 500 мс интервал
- Пропускная способность: 1-2 KB/s
- Максимум клиентов: 4
- Влияние на main loop: нет (non-blocking)

#### Требования
- Встроенные библиотеки ESP32: WiFi, WebServer
- Аппаратный WiFi чип: ESP32-C3 встроенный

### Documentation (Документация)
- Полное руководство по использованию в `WIFI_INTERFACE.md`
- Техническая архитектура в `ARCHITECTURE.md`
- Примеры на Python и JavaScript включены

### Known Issues (Известные проблемы)
- Нет SSL/TLS (HTTP только)
- Нет аутентификации (любой может управлять)
- Нет логирования истории действий

# Документация: Управление платой Kokusan Denki через Arduino Nano

Техническое руководство по запуску шагового двигателя с редуктором с использованием **родной платы управления**, на которой распаяны контроллер частоты **KA20885D** (с кварцевым резонатором) и силовой драйвер **SLA6023**.

---

### 1. Архитектура и преимущества схемы
При использовании родной платы Arduino Nano **не генерирует фазовые сдвиги** самостоятельно. Функцию распределения импульсов и стабилизации частоты берет на себя аппаратный чип **KA20885D**. 

**Плюсы подхода:**
*   **Плавность хода:** Кварцевый резонатор устраняет микро-задержки (джиттер) микроконтроллера. Редуктор работает тише, без лишних вибраций.
*   **Экономия ресурсов:** Процессор Arduino разгружен и может параллельно обрабатывать датчики или дисплей.
*   **Безопасность:** Логика KA20885D аппаратно защищает драйвер SLA6023 от сквозных токов обмоток. Ошибкой в коде сжечь транзисторы невозможно.

---

### 2. Схема подключения (Arduino Nano → Шлейф платы)

Для управления используются стандартные сигналы протокола **STEP / DIR**. 

*   **GND** (Arduino Nano) $\rightarrow$ **GND** (Общий провод интерфейсного шлейфа платы). *Обязательно объединить земли!*
*   **D8** (Arduino Nano)  $\rightarrow$ **DIR** (Выбор направления вращения)
*   **D9** (Arduino Nano)  $\rightarrow$ **CLK / STEP** (Вход тактовых импульсов)
*   **D10** (Arduino Nano) $\rightarrow$ **ENABLE** (Включение/отключение удержания мотора)

*Внимание: Силовое питание (+VCC 12V/24V) для двигателя подается напрямую на плату управления, а не через Arduino.*

---

### 3. Тестовый скетч (Arduino IDE)

Стандартные библиотеки для шаговых двигателей не требуются. Управление происходит через прямую генерацию импульсов.

```cpp
// Определение пинов управления
const int stepPin   = 9;  // Подключается к CLK / STEP на плате
const int dirPin    = 8;  // Подключается к DIR на плате
const int enablePin = 10; // Подключается к ENABLE на плате

void setup() {
  pinMode(stepPin, OUTPUT);
  pinMode(dirPin, OUTPUT);
  pinMode(enablePin, OUTPUT);
  
  // Активация драйвера (если на плате инверсная логика, замените на LOW)
  digitalWrite(enablePin, HIGH); 
}

void loop() {
  // 1. Вращение ВПЕРЕД
  digitalWrite(dirPin, HIGH); 
  
  for(int x = 0; x < 400; x++) {
    digitalWrite(stepPin, HIGH);
    delayMicroseconds(800); // Полупериод импульса (задает скорость)
    digitalWrite(stepPin, LOW);
    delayMicroseconds(800);
  }
  
  delay(1000); // Пауза 1 секунда

  // 2. Вращение НАЗАД
  digitalWrite(dirPin, LOW); 
  
  for(int x = 0; x < 400; x++) {
    digitalWrite(stepPin, HIGH);
    delayMicroseconds(1200); // Вращение с другой скоростью
    digitalWrite(stepPin, LOW);
    delayMicroseconds(1200);
  }
  
  delay(1000); 
}
```

### 4. Особенности настройки при тестировании и запуске

При первом запуске скетча возможны специфические проблемы, связанные с аппаратными ограничениями встроенного контроллера частоты и инерцией редуктора:

*   **Мотор «свистит», гудит, но вал стоит на месте:** 
    Встроенный контроллер с кварцем имеет фиксированный верхний порог частоты, а редуктор обладает высокой инерцией. Если стартовая скорость слишком велика, ротор срывается в резонанс. 
    *   *Решение:* Увеличьте значения задержек в коде `delayMicroseconds` (например, с `800` до `2000` или `3000`), чтобы снизить стартовую частоту шагов. Как только мотор начнет уверенно вращаться, плавно уменьшайте задержку для поиска максимальной скорости.
*   **Двигатель свободно вращается рукой (нет удержания):**
    Сигнал включения на вашей плате может быть инверсным.
    *   *Решение:* Измените состояние пина в блоке `setup` на противоположное: `digitalWrite(enablePin, LOW);`.
*   **Пропуск шагов или сильный нагрев SLA6023:**
    Если на плате присутствуют подстроечные резисторы (потенциометры), они регулируют опорное напряжение и рабочий ток фаз. При тестировании под нагрузкой может потребоваться их точечная корректировка мультиметром.
